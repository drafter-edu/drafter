# Full API Documentation

This full API documentation is provided for those who want to understand the internals of Drafter and potentially contribute to the project. If you are looking for user documentation, please refer to the [Student Documentation](../students/docs.md).

## Module Reference

For detailed API documentation, please see the docstrings in the source code:

- **drafter.server** - Server functionality and configuration
- **drafter.testing** - Testing utilities for Drafter applications
- **drafter.routes** - Route management and URL handling
- **drafter.page** - Page construction and rendering
- **drafter.history** - History and navigation management
- **drafter.deploy** - Deployment utilities
- **drafter.debug** - Debugging tools and information display
- **drafter.configuration** - Configuration management
- **drafter.components** - UI components for building pages
- **drafter.styling** - Styling and CSS utilities

To explore the API in detail, you can:

1. View the source code in the [GitHub repository](https://github.com/drafter-edu/drafter)
2. Use Python's built-in `help()` function after importing Drafter
3. Install Drafter and use your IDE's autocomplete features

Example:
```python
from drafter import *
help(Page)  # Show documentation for the Page class
```
